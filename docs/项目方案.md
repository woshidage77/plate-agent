## 一、项目概述

### 1.1 项目名称
**PlateAgent** — 基于 tRPC-Agent + OpenCV 的车牌图像识别智能体系统

### 1.2 一句话
以 GraphAgent 编排 OpenCV 图像处理流水线，集成 LLM 二次校验与 RAG 知识库，构建可对话、可评测、可服务化的生产级车牌识别 Agent。

### 1.3 三大场景叙事

| 场景 | 讲法 |
|------|------|
| 课设答辩 | 传统OpenCV车牌识别 + 引入AI二次校验提升准确率 + 微服务架构部署 |
| 犀牛鸟评审 | 基于tRPC-Agent的GraphAgent实现车牌识别全链路编排，覆盖 Tool/Session/RAG/Graph/评测/可观测 六大模块 |
| 求职面试 | Spring Boot 管理后台 + tRPC Agent AI引擎 + 异构微服务 + Docker Compose 编排 |

## 三、系统架构

### 3.1 分层结构

```
用户交互层
├── 命令行多轮对话
└── Vue 3 监控大屏 (7.1后)

FastAPI 服务层 (SSE 流式)
├── POST /api/chat              多轮对话
├── POST /api/recognize         单张识别
├── GET  /api/history           历史查询
└── GET  /api/eval/report       评测报告

tRPC-Agent 引擎层 (Python)
├── LlmAgent (对话入口 + 意图路由 + 流式输出)
├── GraphAgent (4节点识别流水线 + 条件路由)
│   ├── [预处理] GaussianBlur + grayscale + binarize + Canny + affine
│   ├── [定位]   形态学 + HSV颜色
│   ├── [分割]   垂直投影
│   └── [识别]   SVM + (conf<0.85? -> LLM校验)
├── Session (Redis) + Memory (跨会话)
├── Knowledge/RAG (ChromaDB 黑名单 + 规范文档)
└── OpenTelemetry tracing

基础设施层
├── Redis      Session + 缓存
├── MySQL      车辆信息 + 识别记录
├── ChromaDB   向量知识库
├── RabbitMQ   异步通知
└── Spring Boot 管理后台 (7.1后)

容器化: Docker Compose 7 服务编排
```

### 3.2 一次完整调用追踪

```
1. 用户: "@识别 plate.jpg" -> LlmAgent 解析意图
2. 流式: "收到图像，开始预处理..."
3. GraphAgent.preprocess -> 流式: "预处理完成"
4. GraphAgent.locate     -> 流式: "车牌已定位"
5. GraphAgent.segment    -> 流式: "字符分割完成，7个字符"
6. GraphAgent.recognize
   |- svm_predict("京") conf=0.92  ok
   |- svm_predict("B")  conf=0.94  ok
   |- svm_predict("2")  conf=0.81  -> LLM校验 -> "2" ok
   |- 结果: "京B12345"
7. RAG查询黑名单 -> 写入Memory -> SSE推送最终结果
8. 用户追问: "上次那辆京A呢？" -> Memory跨会话召回
```

## 八、犀牛鸟 Proposal 提交稿 (200字)

基于 tRPC-Agent 框架构建车牌图像识别智能体系统 PlateAgent。以 GraphAgent 编排预处理、定位、分割、识别四节点流水线，封装高斯滤波、Canny边缘检测、SVM 分类等 10+ FunctionTool 实现模型驱动的工具调用；通过条件路由在 SVM 置信度不足时自动委托 LLM 进行字符二次校验；集成 ChromaDB 构建车辆黑名单 RAG 知识库，支持多轮对话查询与跨会话 Memory 记忆召回；通过 FastAPI + SSE 实现流式服务化，配套 OpenTelemetry 全链路可观测与 SVM/LLM 对比评测体系。项目致力于探索 Agent 框架在计算机视觉工程化场景中的落地路径。
