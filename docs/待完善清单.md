### 2. DeepSeek API 重试机制 → LLM 复核节点无容错

**现状**：项目方案里写了"tenacity 重试 + 降级跳过 LLM"，但 [agent/config.py](/D:/codex_prorject/ai_project/xiniuniaojia/plate-agent/agent/config.py) 和 [agent/graph_nodes.py](/D:/codex_prorject/ai_project/xiniuniaojia/plate-agent/agent/graph_nodes.py) 均未实现。

**为什么缺**：网络波动 / API 限流 → LLM 复核直接抛异常 → `llm_verify_node` 返回空 → 低置信度字符丢失 → 车牌号缺位。

**怎么补**（三层容错）：
```python
# 1. 重试装饰器（tenacity）
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=10))
async def call_deepseek_with_retry(messages, tools):
    return await model.generate(messages, tools)

# 2. 降级策略：重试全部失败 → 用 SVM 原始结果，不阻塞流水线
try:
    llm_result = await call_deepseek_with_retry(...)
except Exception:
    llm_result = svm_result  # 降级兜底

# 3. 超时控制
async with asyncio.timeout(30):
    llm_result = await call_deepseek_with_retry(...)
```

**考试映射**：Agent 工程化鲁棒性（面试级加分项）；错误处理 vs 异常传播（FunctionTool 铁律 4 的延伸）

### 3. Checkpoint + interrupt/resume → 犀牛鸟必考点，零代码

**现状**：Day 10-A 标记为 ⚠️ 概念级。项目中没有实现。

**为什么重要**：犀牛鸟考试 2.2 节明确列了 checkpoint/interrupt/resume。而且这跟你的防幻觉问题是直接相关的——当一个环节出错时，能不能"暂停→人工介入→恢复继续"。

**怎么补**（最小实现，添加到 GraphAgent 流水线）：
```python
# graph_agent.py 中，在 recognize_node 后插入 interrupt
graph.add_node("human_review", human_review_node)
graph.add_conditional_edges(
    "recognize",
    _route_after_recognize,
    {
        "verify": "human_review",     # 低置信度 → 先人工确认
        "output": "format_output",
    },
)

# human_review_node 写一个 interrupt，把结果发给用户确认
async def human_review_node(state, async_writer):
    await async_writer.write_text("⚠ 有低置信度字符，请确认...")
    # interrupt: 暂停执行，等待用户回复
    return {"awaiting_human": True}
```

**考试映射**：GraphAgent 进阶 → interrupt/resume 机制

## 二、中优先级（补足概念级考点）

### 4. Parallel 并行编排 → 项目中有天然应用场景

**现状**：Day 10-A 标记为 ⚠️ 概念级。

**为什么值得加**：你的 `recognize_node` 里对每个字符逐个 SVM 识别是串行的——但 7 个字符互不依赖，天然可以并行。用 Parallel Agent 做字符并行识别，既是真实优化，又是考试亮点。

**实现思路**：
```python
# 当前（串行）：for char_path in char_images: svm_predict(char_path)
# 改为（并行）：ParallelAgent 同时调 7 个 Worker
parallel_agent = ParallelAgent(
    workers=[SVMWorker(char_path) for char_path in char_images],
    merge_strategy="concat"  # 按原顺序拼接结果
)
```

**考试映射**：Chain/Parallel/Cycle → Parallel 真实案例

目前 OTel 只有 ConsoleSpanExporter（本地打印），接 Langfuse 可以可视化 trace、看 token 消耗趋势、做 prompt 版本对比。

当 LLM Judge 评出低分案例 → 自动调 prompt 或调阈值 → 重新评测 → 对比提升。这是犀牛鸟 proposal 级亮点。
