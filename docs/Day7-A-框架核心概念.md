# PlateAgent Day 7-A：评测体系框架知识（考试向）

> 源：ML评测方法论 + 项目实战
> 用途：7.1 犀牛鸟考试知识储备

---

## 一、为什么需要评测体系

### 1.1 没评测的问题

改了一行预处理代码，准确率是升了还是降了？不知道。

换了新的 Embedding 模型，黑名单命中率有没有变差？不知道。

### 1.2 评测体系的答案

```测试集（30张合成车牌 + ground truth）
  ↓
批量跑识别流水线
  ↓
对比预测 vs 标注
  ↓
量化报告：准确率 / 字符准确率 / 按条件分组 / 耗时
```

**有了评测，每次改动后跑一遍就知道效果。**

---

## 二、评测指标

### 2.1 整体准确率

完全匹配才算对：

```accuracy = correct_count / total_count
```

### 2.2 字符级准确率

每个位置分别判断：

```char_accuracy = sum(each_position_correct) / total_characters
```

适用场景：SVM 占位时整体 0%，但字符级能看出部分改善。

### 2.3 按条件分组

| 条件 | 样本数 | 说明 |
|------|--------|------|
| clear | 10 | 清晰标准图（baseline） |
| blur | 10 | 高斯模糊（模拟运动模糊） |
| tilt | 5 | 旋转 ±5~15°（模拟倾斜拍摄） |
| noise | 5 | 椒盐噪声（模拟低光照） |

---

## 三、评测数据生成

### 3.1 PlateAgent 的策略

真实车牌图片 + ground truth 获取困难，用 PIL 合成：

- 蓝底白字标准车牌格式
- 覆盖 30 个不同省份的车牌号
- 4 种干扰条件随机施加

### 3.2 生成参数

| 条件 | 参数范围 |
|------|---------|
| blur | GaussianBlur radius 0.3~1.0 |
| tilt | rotate -15°~+15° |
| noise | 椒盐噪声 2%~10% 像素 |

---

## 四、考试速记卡

| 考点 | 答案 |
|------|------|
| 整体准确率 vs 字符准确率？ | 整体=完全匹配，字符=逐位判断 |
| 评测的四个条件分组？ | clear / blur / tilt / noise |
| PlateAgent 评测集大小？ | 30 张（10+10+5+5） |
| ground truth 格式？ | JSON [{id, image, plate_number, conditions}] |
| 评测报告包含什么？ | 整体准确率、字符准确率、按条件分组、逐张详情、耗时 |
| 为什么用合成图？ | 真实车牌数据涉及隐私，合成图可控变量 |
