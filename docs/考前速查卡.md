| 抽象 | PlateAgent 文件 | 一句话 |

## 一次调用的完整链路

```
用户输入 → Runner.run_async()
  → SessionService.get() 恢复历史
  → MemoryService.search() 召回记忆
  → Agent 构造 messages
  → Model.generate() → tool_calls?
  → Tool.execute() → 结果回传
  → Model.generate() → 最终回复
  → 事件流输出 → SessionService.append() 持久化
```

## FunctionTool 工作原理

模型**不直接调用** Python 函数：
1. 框架发 tools schema（name + description + parameters）
2. 模型返回 `tool_calls: [{name, arguments}]`
3. 框架按 name 找到函数，用 arguments 调用
4. 返回值作为 tool_result 发回模型

## Session vs Memory

- **Session**: 当前对话上下文（消息/事件），短期，对话结束可丢
- **Memory**: 跨会话长期记忆（用户偏好/历史），持久化，支持检索

## RAG 五步

文档 → 分块 → 向量化(embedding) → 存 ChromaDB → 检索 → 拼 prompt → 生成

## OTel: Trace vs Span

- **Trace**: 一次完整请求（如识别一张车牌的全流程）
- **Span**: 流程中的一个节点（如 preprocess 耗时 83ms）
- **ConsoleSpanExporter**: 开发用，打印 JSON
- **OTLPSpanExporter**: 生产用，发 Jaeger/Grafana

## LLM Judge vs 精确匹配

- **精确匹配**: predicted == ground_truth（Day 7）
- **LLM Judge**: DeepSeek 按评分标准打分，容忍 OCR 混淆（Day 8）
- 关键: 评分标准写死 prompt，temperature=0，JSON 约束输出

## 三层容错

```
asyncio.timeout(30s) → tenacity retry(3次, 指数退避) → fallback 降级
可重试: TimeoutError/ConnectionError/429/502/503
不重试: 401/403 (API key 错误)
```

## Interrupt / Resume

```python
from trpc_agent_sdk.dsl.graph import interrupt
human_response = interrupt({"message": "请确认字符"})
# 暂停 → 等用户回复 → 继续
```

Checkpoint: interrupt 前自动保存 state 快照，resume 时恢复。

## 防幻觉四道闸门

```
conf >= 0.85 → 直接通过
0.5 <= conf < 0.85 → LLM 复核 (retry+降级)
LLM 全失败 → SVM 兜底
conf < 0.5 → interrupt 人工确认 + [?] 标注
```
